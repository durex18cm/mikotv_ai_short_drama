# Miko TV

### 把一个创意，变成一部成片短剧的 AI Agent 工作流

**创意进，短剧出。** 上传一张图片或一句创意，一组 AI Agent 协同写剧本、立人设、拆分镜、出关键帧、生成视频，再配上配音、字幕、音乐音效，最终剪成一条可直接发布的竖屏短剧。

## ✨ 为什么需要 Miko TV

如今做一部 AI 短剧，往往要在一堆互不相通的工具间来回切换——写剧本一个、画人设一个、图生视频一个、配音一个、剪辑又一个。每一次交接都会丢失上下文，角色和风格的一致性也在中途崩塌。

**Miko TV 把整条流水线收拢成一个被引导的 Agent 工作流。** 你只做真正重要的创作决策——*创意、角色、风格*——其余全部交给一组协同的 Agent 接管，让角色、风格、调性从第一帧到最终成片保持一致。

> **核心理念：** *用户少操作，AI 多完成。*
> 一条 AI 短剧创作的端到端闭环，而不是又一个笨重的专业剪辑软件。

| 现在（自己拼） | 用 Miko TV |
| --- | --- |
| 6–10 个互不相通的工具 | 一条连续的工作流 |
| 自己写提示词、调模型参数 | 只挑创意、角色和风格 |
| 角色在镜头间前后不一致 | 角色设定全流程强约束 |
| 一个镜头不好就重做整集 | 单个镜头 / 台词 / 配音局部重做 |
| 几小时的手动剪辑 | 自动合成成片 |

## 🎬 它能做什么

给平台一张参考图（可选）和一个短剧类型，它便会跑完一整套创作闭环：

```
创意 / 图片  →  剧本  →  角色  →  风格  →  分镜
     →  关键帧  →  视频  →  配音 + 字幕  →  自动剪辑  →  预览  →  导出
```

你真正要动手的只有 6 步：

**① 描述** → **② 确认角色** → **③ 选择风格** → **④ 一键生成** → **⑤ 局部修改不满意的镜头** → **⑥ 导出**

其余的一切——场景拆分、镜头拆解、提示词生成、关键帧、渲染、声音匹配、音乐音效、成片合成——都在背后自动完成。

## 🧩 Agent 工作流（12 个阶段）

每个阶段由一个专职 Agent 驱动。它们共享同一份项目上下文，因此下游每个阶段都建立在剧本、角色设定和所选风格之上。

| # | 阶段 | Agent 做的事 |
| --- | --- | --- |
| 1 | **创建项目** | 设定作品名称、剧集数、时长、画面比例（9:16）、语言 |
| 2 | **选择短剧类型** | 选类型 + 高频细分 + 核心爽点——无需写剧本 |
| 3 | **剧本解析** | 自动拆分为剧集、场景、镜头、台词、旁白、动作 |
| 4 | **确认角色** | 自动识别角色并生成设定（外貌、服装、性格、声音） |
| 5 | **视觉风格** | 全片统一风格（写实电影 / 国风史诗 / 动漫 / 游戏 CG / 水墨 / 竖屏短剧） |
| 6 | **生成分镜** | 基于剧本 + 角色 + 风格生成逐镜头分镜表 |
| 7 | **生成关键帧** | 每个镜头的视觉草稿——满意的可锁定后再渲染 |
| 8 | **生成视频** | 通过可插拔视频模型，从锁定的关键帧渲染每个镜头 |
| 9 | **配音与字幕** | 自动匹配旁白与角色配音，自动生成字幕 |
| 10 | **剪辑合成** | 拼接片段、对齐配音、添加字幕、音乐、音效、转场 |
| 11 | **成片预览** | 观看成片；可就地重做任意单个镜头 / 台词 / 配音 |
| 12 | **导出视频** | 竖屏 9:16 MP4，可选带字幕 / 不带字幕 |

### 设计原则

- **创意是唯一必填输入。** 不写提示词、不调模型参数、不碰时间轴。
- **只确认真正重要的：** *剧本对不对？角色对不对？风格对不对？*
- **先关键帧，后视频**——用更低成本先确认画面，再付费渲染。
- **局部重做，而非整片重做**——修一个镜头，不必重生成整集。
- **每个阶段状态清晰**——未开始、排队中、生成中、已生成、失败、已确认、已锁定。

## 🎥 可插拔的视频模型

视频生成与具体模型解耦。每个镜头都通过统一的适配层渲染，因此你可以按风格、预算或可用性，把镜头路由到最合适的引擎——更换厂商不影响流水线的其余部分。

| 模型 | 优势 |
| --- | --- |
| **Google Veo** | 电影级运镜，提示词遵循强，单镜头更长更连贯 |
| **Seedance** | 快速、富表现力的图生视频，擅长风格化与角色驱动的镜头 |
| **HappyHorse** | 高性价比批量渲染，适合大批量剧集生产 |

> 同样的适配模式贯穿整个 AI 层——LLM、图像、配音与音乐厂商都在稳定接口背后可随时替换。

## 🛠️ 技术栈

**前端**
- React 18 · TypeScript 5 · Vite 5
- Tailwind CSS 3 · shadcn/ui（Radix UI）
- framer-motion · lucide-react

**后端与 API**
- Python 3.11+ · FastAPI · 异步 I/O · Pydantic
- WebSocket 流式推送实时生成状态

**Agent 层**
- 基于共享项目上下文图的多 Agent 编排
- 各阶段专职 Agent（剧本、角色、分镜、关键帧、视频、配音、音乐、剪辑）
- 工具/函数调用、结构化输出，以及支持局部重做的逐阶段重试

**AI 模型**
- **LLM** —— 剧本解析、角色设计、提示词合成
- **图像** —— 带角色与风格约束的关键帧生成
- **视频** —— 可插拔引擎：**Google Veo**、**Seedance**、**HappyHorse**
- **配音** —— 旁白与台词的 TTS 及声音匹配
- **音乐音效** —— 生成式配乐与音效匹配

**流水线与基础设施**
- 分布式任务队列，承载长耗时生成任务
- 对象存储托管素材（关键帧、片段、音频、成片）
- PostgreSQL 存储项目与素材元数据
- 基于 FFmpeg 的自动剪辑与成片合成
- Docker 保证可复现部署

## 🚀 快速开始

Web 前端：

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview
```

随后打开 Vite 输出的本地地址（默认 `http://localhost:5173`）。
